# Story 2.6: Last Player option — Swap or draw from the Deck

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As the Last Player,
I want a third choice — draw a random Card from the Deck instead of swapping,
so that the final seat (and the heads-up game) has the luck-of-the-draw escape.

## Acceptance Criteria

> Source ACs verbatim from [epics.md#Story 2.6]. The "**Then**" clauses are the binding contract; the AC IDs are this story's addressing scheme. This story closes Epic 2: it adds the third turn-scoped intent (`drawFromDeck`), computes the real `isLastPlayer`, and emits the `allActed` phase that hands off to Epic 3 / Story 3.2.

**AC-2.6.1 — The Last Player (and ONLY that seat) sees a subordinate "Draw from deck" Secondary button alongside SWAP/KEEP (FR-7, UX-DR5)**
Given the single Last Player (whose right-hand neighbor is the Starting Player),
When their Turn arrives,
Then their device adds a **visually-subordinate Secondary "Draw from deck" button** alongside SWAP/KEEP — shown **ONLY on that one seat**; every other Player still sees **exactly Swap/Keep**.

**AC-2.6.2 — Drawing replaces the Card with a random deck Card; the discarded Card leaves the Deck for the rest of the Round (FR-7)**
Given the Last Player draws from the Deck,
When the draw resolves (carrying the turn token),
Then their Card is **replaced by a random Card from the remaining Deck**, the **discarded Card is removed from the Deck for the rest of the Round** (returns at the next Deal), and **this is the final Turn of the one pass**.

**AC-2.6.3 — The final accepted turn action transitions the round to the `allActed` phase (Winston review)**
Given the last action of the one-pass resolves (Last Player's Swap/Keep/draw),
When the turn completes,
Then the round phase transitions to the **`allActed` phase value** (the phase named in the 1.3 contract) — so Story 3.2's `revealAll` only **READS/consumes** it and never introduces it.

**AC-2.6.4 — There is ALWAYS ≥1 Card available to draw — deck size ≥ player count by construction (edge E2)**
Given the deck-draw availability (edge E2),
When the Last Player draws,
Then there is **ALWAYS ≥1 Card available** — deck size ≥ player count by construction (52 ≥ 20 single-deck; 104 for 11–20 two-deck), so **no empty-deck path exists**; this is stated so a reviewer need not re-derive it and a future change can't silently break it.

**AC-2.6.5 — Heads-up (2-Player) path: Player 2 (Last Player) may Swap with Player 1 OR draw (pure luck) (FR-7 heads-up path)**
Given a heads-up Table (2 Players),
When the Round plays,
Then **Player 1 (Starting Player)** may Swap (exchange with Player 2) or Keep; **Player 2 (Last Player)** may Swap with Player 1 or **draw from the Deck** — the deck draw is **pure luck**.

### The accepted-path chokepoint — handleDraw is a turn handler that PERSISTS
The three gameplay precedents (read them — handlers.ts):
- `handleDeal` (PHASE scope): shape → table-null → not-host → **checkPhaseToken** → phase → ≥2-alive → assertDealable → mutate → **bumpPhaseToken** → **persist**. [handlers.ts:331]
- `handleSwap`/`handleKeep` (TURN scope): `requireActiveTurn` (shape → table-null → round-null/phase → not-your-turn → **checkTurnToken**) → applyX → **bumpTurnToken** → **NO persist** (memory-only round change, phase stays `turns`). [handlers.ts:472, 491]
- **`handleDraw` (TURN scope, but phase-changing):** `requireActiveTurn` → **last-player check** → applyDraw → set `phase = "allActed"` → **bumpTurnToken** → **bumpPhaseToken** → **persist**. It is the hybrid: turn-scoped entry (turnToken guard) but it advances the PHASE (so it bumps the phase token AND persists, like a deal). This is the ONE turn handler that persists. Spell this out in its JSDoc so a reviewer doesn't flag the persist as a mistake.
- **Why persist:** `persistSummary`/`toSummary` keeps `{code, phase, hostId, startingLives, players, phaseToken}` and drops `round` [persistence.ts]. A draw changes `phase` (turns→allActed) and `phaseToken` — BOTH durable. `persistence.ts` LIVE_ROUND_PHASES already includes `allActed` [persistence.ts:63], so a DO reload mid-`allActed` coerces correctly (D2.1). Swap/keep change neither (phase stays `turns`), hence their no-persist.

### Where `allActed` is decided — the LAST seat completes the pass (any action, not just draw)
- AC-2.6.3 says the trigger is the **"Last Player's Swap/Keep/draw"** — i.e. the final accepted turn action of ANY kind. So the `turns → allActed` transition is NOT exclusive to `drawFromDeck`; `handleSwap`/`handleKeep` must ALSO transition when the actor was the last to act. ### The randomness lives in the SHUFFLE, not in `applyDraw` (purity boundary)
- The deck in `round.deck` was already Fisher–Yates shuffled by `dealRound` (seeded by `cryptoRng()` outside the engine) [engine.ts:110; rng.ts].
