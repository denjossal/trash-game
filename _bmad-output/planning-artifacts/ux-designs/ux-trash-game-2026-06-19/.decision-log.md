- **Product:** Real-time, in-person party card game. Web app, no download, no accounts. Players 2-20 on their own phones at one physical table.
- **Stakes:** Hobby / personal — "made for the maker's own people."

## Load-bearing constraints from the PRD (these drive the UX)

- **Eyes up, not down** (PRD §10): the defining discipline. App is a quiet dealer + scorekeeper; ~10 sec of screen attention per round. Doing *less* on screen on purpose.
- **Two design gates:** Host (activation — start in seconds) and Reluctant Player (weakest-link — two big buttons, zero confusion).
- **Two co-equal beats:** the squirm (Swap decision) and the simultaneous reveal (Showdown "OHHH").
- **Active Player screen = exactly two big buttons** (Swap / Keep) + peek/hide. Inactive screens = only whose Turn it is.
- **Hard privacy rule:** a secret Card is shown only to its owner until Showdown — not in UI, not in traffic, not inferable from behavior. Card hidden by default, peek requires explicit hold, auto-re-hides.
- **Comedy FX deferred to v1.1.** MVP keeps a minimal-but-real simultaneous-reveal beat.
- **Large tables (up to 20):** app computes & highlights Loser(s); humans don't eyeball 20 cards.

## Decisions

- **Working mode:** Design handoff. Assemble a producer-shaped prompt (default target: Google Stitch) for the external tool to generate visual design + screens. EXPERIENCE.md follows via Update mode once visuals return. (2026-06-19)
- **Visual personality:** Undecided by label — "let me suggest." Handoff prompt instructs the tool to explore **2-3 distinct directions** (e.g., bold playful party / clean minimal / warm tactile card-table) so Dennis picks from rendered output rather than from a label.
- **Color scheme:** **Dark-mode default** (dim party lighting; less of an attention-pulling beacon). Light mode optional later.
- **Form-factor:** **Phone-only.** Every player incl. Host on their own phone. No shared screen — matches PRD non-goal. Portrait orientation assumed `[ASSUMPTION]`.
- **Screens to generate:** Core loop (Home/Create-or-Join, Lobby, Your-Turn, Waiting, Showdown, Eliminated/Winner) + Host conductor controls (FR-14 + deal/showdown/re-deal triggers) + Peek/hide card interaction (privacy-critical, tactile) + Error/edge states (bad room code, disconnect, empty lobby, all-tied showdown).
- **Card visual style:** **Big value + suit, simplified** — huge rank + suit pip, readable across a table; not photoreal, not tiny corner index.

- **CONFLICT — tone drift.** Stitch `manifest.json` describes Trash as *"A high-stakes underground card gathering for friends"* and DESIGN.md Brand & Style voice is "loud, unapologetic." This DRIFTS from the PRD's warm, good-natured, family/Abuela-and-kids, comedy-not-cruelty positioning (PRD §10). **RESOLUTION (user):** keep the visuals, **rewrite the voice/microcopy to the PRD's warm tone**; drop "high-stakes underground." Neon = "party energy," not "edgy nightclub." Manifest description to be corrected at finalize.

(Did not pick "dim the between-turn screens" as a hard rule, but Waiting remains the calmest surface by element-count.)

### Finalize — reviewer gate + resolutions (2026-06-19)

Three lenses run in parallel (full outputs: `review-accessibility.md`, `review-fidelity.md`, `review-rubric.md`). All strong verdicts; no blockers, no contradictions, PRD fidelity confirmed (King social-only honored — no FR-8 regression; all game rules accurate; 3 UJs mirrored as flows).

Resolutions applied:
- **Token footgun (rubric LOW, build-critical):** prose said "Secondary Neon Mint" for CTAs but `secondary`=#ffffff; mint is `secondary-container` #36ffc4. Fixed all refs + added a token note. Primary button = mint on black (15.6:1).
- **Lives pip contrast (a11y HIGH H-1):** spent pip was `outline-variant` #514255 (2.0:1, fails). Changed to `outline` #9d8ba0 + shape distinction (filled vs ring).
- **Motion/flash safety (a11y HIGH H-2/H-3):** added DESIGN Motion & Flash section — default flip ≤400ms, no strobe, no flash >3×/s, no full-viewport flash; Reduce Motion extended to pulse + scale. Turn-frame pulse scoped to Your Turn only (Waiting stays static/calm).
- **Receded-text floor (a11y HIGH H-4):** non-losing showdown cards dim to no lower than 70% opacity (rank stays ≥4.5:1).
- **Screen-reader peek path (a11y MED M-4):** added announce-once-and-discard owner-only path; hidden rank absent from a11y tree.
- **Host conductor bar visual spec (rubric HIGH):** added DESIGN component (bottom thumb-zone bar + controls overlay; absent on Your Turn / for non-Hosts).
- **Eliminated surface in no flow (rubric HIGH):** added Flow 5 (spectator/heckle-from-sidelines).
- **Last-Player 3rd button vs two-button rule (rubric MED):** reconciled in State Patterns — bounded single-seat exception, Secondary-styled, every other Player still sees only Swap/Keep.
- **Showdown flip undefined (rubric MED):** now defined in DESIGN Motion & Flash; EXPERIENCE references it.
- **Host-starts-first-Round (fidelity HIGH):** added to Round Result state row.
- **No-join-in-progress (fidelity MED):** added to Joining state row.
- **FX deferred to v1.1 (fidelity MED):** added to Showdown state row.
- **Manifest.json (fidelity MED):** the actual artifact fixed — removed stray Stitch path line (was invalid JSON), rewrote "high-stakes underground" → warm description, corrected background/theme color #170f24 → #1a0b2e (chosen base).
